= Syngrafo Editor Package
:toc: left
:toclevels: 2

== Overview

`@syngrafo/editor` is a self-contained React component library that provides a
block-based document editor built on the Syngrafo Document Model (SDM).
== Architecture

----
EditorShell          top-level component — context tabs, panels, header, PDF export
  EditorProvider     zustand-free useReducer store (editor-store.tsx)
  EditorCanvas       document canvas — renders blocks, page rulers, @page CSS
  Toolbar            block type / align / style controls + undo / redo
  BlockView          per-block renderer (text, img, layout, table, callout …)
  DocumentPanel      page size / margin / background / page-numbers settings
  StylePanel         named style classes + per-block overrides
  BlockTreePanel     document outline with drag-to-reorder
  StatsPanel         word / char / readability stats
  NLPPanel           NLP annotation overlay controls
  ExportPanel        Files tab — templates, recent docs, save/export, import
----

== Syngrafo Document Model (SDM)

=== Document envelope

[source,json]
----
{
  "$schema": "syngrafo/1",
  "id":      "<uuid-v4>",
  "meta": {
    "title":      "My Document",
    "created_at": 1700000000,
    "updated_at": 1700000000,
    "zone":       null,
    "author":     null,
    "tags":       []
  },
  "page": {
    "size":            "a4",
    "orientation":     "portrait",
    "margin":          "md",
    "background":      { "color": "#faf7f2" },
    "showPageNumbers": false
  },
  "styles": {
    "display-title": { "props": { "font": "serif", "size": "3xl", "weight": "bold" } }
  },
  "blocks": []
}
----

`background.color` accepts any CSS color string.  Defaults to white when absent.
Set `"transparent"` for overlay / screencast use.
Printed faithfully via `-webkit-print-color-adjust: exact`.

NOTE: Images are *never* stored as base64 inside the document.
They are referenced by `src` (`asset://filename`, `local://abs-path`, or `https://…`).
Base64 is only produced at PDF/HTML export time.

=== Page sizes

`a0`–`a6`, `letter`, `legal` — all stored as labels and resolved to mm at render time
via `PAGE_SIZE_MM` in `models/sdm.ts`.

=== Block catalog

[cols="1,1,3"]
|===
| Type | Category | Notes

| `p` | Text | Paragraph
| `h1` `h2` `h3` `h4` | Text | Headings
| `quote` | Text | Block quotation
| `li` | Text | List item (child of `ul`/`ol`)
| `td` `th` | Data | Table cell / header cell
| `figcaption` | Text | Image caption
| `ul` `ol` | List | Contains `li` children
| `code` | Verbatim | `text` string + `language` hint
| `img` | Media | `src`, `alt`, `fit` (contain/cover/fill), `caption`
| `hr` | Structural | Thematic divider
| `pagebreak` | Structural | Forces a new PDF page
| `callout` | Semantic | `variant` (info/tip/warning/danger/success/note), `title`, `children`
| `hbox` | Layout | Horizontal flex container — direct children must be `col`
| `vbox` | Layout | Vertical flex container — children can be any block
| `col` | Layout | Column inside `hbox`/`grid`.  Has `width` and `span`
| `grid` | Layout | CSS-grid container.  `columns` defines track sizes
| `table` | Data | Rows (`tr`) → cells (`td`/`th`).  Optional `data_source`
| `tr` | Data | Table row
|===

=== Style properties (`SStyleProps`)

Token-based — no raw CSS ever written by the author.

`font` (serif/sans/mono) · `size` (xs–4xl) · `weight` (normal/medium/semibold/bold) ·
`style` (normal/italic) · `color` (hex/rgba) · `background` (hex/rgba) ·
`leading` (tight–loose) · `tracking` (tight/normal/wide/wider) ·
`align` (left/center/right/justify) · `decoration` (none/underline/line-through) ·
`border` { color, width, side, radius } · `spacing` { inner, outer }

=== Data sources (table `data_source`)

[source,json]
----
{ "type": "csv",   "path": "asset://data.csv" }
{ "type": "xml",   "path": "asset://data.xml", "xpath": "//row" }
{ "type": "query", "zone": "invoices",          "sql": "SELECT name, date FROM items" }
----

== Features

=== Undo / Redo

Full undo/redo history via `models/history.ts` — functional pure stack, 50-entry cap.
Wired to `editor-store.tsx` actions: `UNDO`, `REDO`.
Keyboard: `⌘Z` / `⌘⇧Z`.  Toolbar buttons with disabled state at boundaries.

=== PDF Export

`handleExportPDF` in `EditorShell.tsx`:

1. If no canvas context is active, switches to `review` temporarily.
2. Adds `body.sgf-exporting` class (hides all `.sgf-ui` chrome).
3. Waits two animation frames for React to commit the DOM.
4. Calls `dms_save_pdf(uuid, title, zone, path, width_mm, height_mm, orientation)`.
5. Restores previous context.

Page dimensions are passed from `doc.page` (mm), converted to inches in C++.
`@page { size: …mm }` CSS is injected by `EditorCanvas` and respected on all backends:
macOS (WKWebView NSPrintOperation), Windows (WebView2 PrintToPdf),
Linux (WebKitGTK webkit_print_operation_print), Qt (QWebEnginePage::printToPdf).

`showPageNumbers` injects `@bottom-right { content: counter(page) }` — requires
WebKit ≥ 2022 or Chromium ≥ 119; silently omitted on older engines.

=== Page Background

`page.background.color` — any CSS color string.  Renders as `backgroundColor` on the
canvas page div with `print-color-adjust: exact` so it appears in PDFs.
Presets: White / Warm Paper / Cream / Slate / Night / Ink / Transparent.
Custom hex/rgba supported via DocumentPanel color picker.

=== .sdoc Bundle Format

A `.sdoc` file is a plain ZIP archive:

----
document.json          SDM document (syngrafo/1)
assets/<filename>      embedded images and media
----

Load: `dms_fetch_data_url` → `loadSdocBundle()` (fflate unzip in JS) → blob store → render.
Save: `saveSdocToPath()` (fflate zip in JS) → `dms_write_base64_file` → disk.
Save-as: `dms_select_save_path()` native Save dialog → write → `dms_record_export` audit.

=== Built-in Templates

Stored as SDM JSON files in `data/documents/<category>/`:

[cols="1,1,2"]
|===
| Folder | File | Description

| `freeform/` | `blank.json` | Empty A4 document
| `letter/` | `personal-letter.json` | Personal correspondence layout
| `invitation/` | `event-invitation.json` | Event card (A5, centered)
| `invoice/` | `invoice.json` | Line-item invoice with totals table
| `report/` | `quarterly-report.json` | KPI strip + findings + appendix table
| `cv/` | `resume.json` | Two-column CV (sidebar + main content)
| `notes/` | `notes.json` | Bullet notes + action items + callout
| `book/` | `chapter.json` | A5 book chapters with page-break
|===

Displayed in the Files panel (ExportPanel) via `dms_scan_dir`.
Opening a template assigns a fresh document UUID so it never overwrites the template.

== IPC Binding Surface

[source]
----
dms_document_save    uuid, title, blocks_json, styles_json, page_json, zone_name
  → { ok, data: { uuid, updated_at } }

dms_document_load    uuid
  → { ok, data: { uuid, title, blocks_json, styles_json, page_json, zone_name, … } }

dms_document_list    zone_name, limit
  → { ok, data: [{ uuid, title, zone_name, created_at, updated_at }] }

dms_save_pdf         uuid, title, zone_name, export_path, width_mm, height_mm, orientation
  → { ok, data: { path, exported_at } }

dms_save_html        uuid, title, html_content, zone_name, export_path
  → { ok, data: { path, file_size, exported_at } }

dms_select_save_path suggested_name, filter_ext
  → { ok, data: { path } }   (path="" if user cancelled)

dms_record_export    path, doc_uuid, title, zone_name, kind
  → { ok, data: { path, exported_at, file_size } }

dms_get_recent_exports limit
  → { ok, data: [{ id, doc_uuid, title, path, kind, zone_name, exported_at, file_size }] }

dms_write_base64_file  path, base64_data
  → { ok, data: { written, bytes } }

dms_fetch_data_url     path
  → { ok, data: { data_url } }

dms_scan_dir           path, recursive
  → { ok, data: { path, items: [{ name, path, is_dir, size, mtime }] } }

dms_open_path          path_or_uri
  → { ok, data: { opened } }
----

== Hooks

[cols="1,3"]
|===
| Hook | Purpose

| `useAssetSrc(src)` | Resolves `asset://` / `local://` → data URL via IPC or blob store
| `useClickOutside(ref, cb, enabled)` | Click-outside dismissal
| `useDocumentStats(doc)` | Block / word / char counts
| `useNLPAnalyze(canRun)` | NLP analysis pipeline with debounce
| `useIsNarrow(breakpoint)` | Reactive screen-width check
| `useSwipeToPan(ref, onLeft, onRight)` | Touch swipe gesture for panel drawers
|===
