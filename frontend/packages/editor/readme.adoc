== Naming

[cols="1,3"]
|===
| Term | Meaning

| *SDM*
| Syngrafo Document Model — the canonical in-memory and on-disk representation.
  File extension: `.sdoc`.

| *Block*
| A single node in the document tree.
  Every block has an `id`, a `type`, optional `style` class, and type-specific fields.

| *Span*
| An inline text run inside a text block.  Carries marks (`bold`, `italic`, …)
  and an optional `style` class for colour/font overrides.

| *Style class*
| A named, reusable set of typed properties defined in `doc.styles`.
  Applied via `"style": "class-name"` on any block or span.
  The author never writes raw CSS; the renderer resolves classes at export time.

| *Import*
| `HTML → SDM`  Parses an HTML document into a block tree.

| *Render*
| `SDM → HTML`  or  `SDM → PDF`  — the output pipeline.
|===


== Scheme Draft

=== Document envelope

[source,json]
----
{
  "$schema": "syngrafo/1",
  "id":      "<uuid-v4>",

  "meta": {
    "title":      "",
    "created_at": 0,
    "updated_at": 0,
    "zone":       null,
    "author":     null,
    "tags":       []
  },

  "page": {
    "size":        "a4",
    "orientation": "portrait",
    "margin":      "md"
  },

  "styles": {
    "display-title": { "font": "serif", "size": "3xl", "weight": "bold", "color": "#1a1a1a" },
    "subtitle":      { "font": "sans",  "size": "lg",  "weight": "normal", "color": "#555" },
    "accent-rule":   { "color": "#c00", "width": "2px" }
  },

  "blocks": []
}
----

NOTE: Images are *never* stored as base64 inside the document.
They are referenced by `src` (asset UUID, relative path, or `local://` URL).
Base64 is only produced at PDF/HTML export time.


=== Block catalog

[cols="1,1,3"]
|===
| Type | Category | Notes

| `p` | Text | Paragraph.  Contains `spans[]`.
| `h1` `h2` `h3` `h4` | Text | Headings.  Contains `spans[]`.
| `quote` | Text | Block quote.  Contains `spans[]`.
| `li` | Text | List item.  Contains `spans[]`.
| `ul` `ol` | List | Contains `li` children.
| `code` | Verbatim | `text` string + `language` hint.  Monospace, no spans.
| `img` | Media | `src`, `alt`, `fit` (`contain`/`cover`/`fill`).
| `hr` | Structural | Thematic divider.  Style class controls colour + thickness.
| `pagebreak` | Structural | Forces a new PDF page.
| `hbox` | Layout | Horizontal flex container.  Children are `col` blocks.
| `vbox` | Layout | Vertical flex container.  Children are any block.
| `col` | Layout | Column inside `hbox` or `grid`.  Has `width` and `span`.
| `grid` | Layout | CSS-grid container.  `columns` defines track sizes.
| `table` | Data | Rows → `tr`, cells → `td`/`th`.  Future: `data_source`.
| `tr` | Data | Table row.  Children are `td`/`th`.
| `td` `th` | Data | Table cell.  Contains `spans[]`.
|===


=== Block node — shared fields

[source,json]
----
{
  "id":   "<uuid>",
  "type": "<see catalog>",

  "style":   "class-name-or-null",

  "spacing": {
    "inner": "none | xs | sm | md | lg | xl | 2xl",
    "outer": "none | xs | sm | md | lg | xl | 2xl"
  },

  "align": {
    "h": "start | center | end | fill",
    "v": "top   | middle | bottom | fill"
  },

  "children": []
}
----

`spacing.inner` maps to CSS `padding` at render time. +
`spacing.outer` maps to CSS `margin`.  +
Authors never see pixels — only the scale token.


=== Spacing and alignment as UI abstractions

[cols="1,3"]
|===
| UI control | Maps to

| `[ ← ]  [ ↔ ]  [ → ]  [ ↕ ]`
| `align.h`: `start`, `center`, `end`, `fill`

| `[ ↑ ]  [ ↕ ]  [ ↓ ]  [ ↕ ]`
| `align.v`: `top`, `middle`, `bottom`, `fill`

| Spacing slider / token picker `xs … 2xl`
| `spacing.inner` or `spacing.outer` depending on selection context
|===

The Property Panel shows these controls for the *selected block*.
Alignment set on a *container* (`hbox`/`vbox`/`grid`) applies to all its children. +
Alignment set on a *leaf block* positions content inside itself (e.g. centred image).

Margin/padding are never exposed directly.


=== Text block — inline spans

[source,json]
----
{
  "id":    "p-1",
  "type":  "p",
  "style": "body",
  "spans": [
    { "text": "Hello " },
    { "text": "world",   "marks": ["bold"] },
    { "text": "!",       "marks": ["bold", "italic"] },
    { "text": "link",    "marks": ["link"], "href": "https://example.com" },
    { "text": "accented", "style": "accent" }
  ]
}
----

Marks: `bold`, `italic`, `underline`, `strike`, `code`, `link`, `sup`, `sub`.
A span `style` class overrides colour/font for that run only.


=== Layout containers — hbox / vbox / grid / col

[source,json]
----
{
  "id":    "header-layout",
  "type":  "hbox",
  "gap":   "lg",
  "align": { "v": "middle" },
  "children": [
    {
      "id":    "logo-col",
      "type":  "col",
      "width": "33%",
      "children": [
        {
          "id":  "logo",
          "type": "img",
          "src":  "asset://logo-uuid",
          "fit":  "contain",
          "align": { "h": "center", "v": "middle" }
        }
      ]
    },
    {
      "id":    "text-col",
      "type":  "col",
      "width": "67%",
      "align": { "v": "middle" },
      "children": [
        { "id": "title",   "type": "h1", "style": "display-title",
          "spans": [{ "text": "VHC Einladung" }] },
        { "id": "rule",    "type": "hr", "style": "accent-rule" },
        { "id": "subtitle","type": "p",  "style": "subtitle",
          "spans": [{ "text": "Herzlich willkommen" }] }
      ]
    }
  ]
}
----

`grid` is the same shape but uses `columns` instead of per-`col` widths:

[source,json]
----
{
  "type":    "grid",
  "columns": ["1fr", "2fr", "1fr"],
  "gap":     "md",
  "children": [
    { "type": "col", "span": 1, "children": [] },
    { "type": "col", "span": 2, "children": [] }
  ]
}
----

Use `hbox` for simple two- or three-column layouts (header, sidebar). +
Use `grid` when you want a Layout Builder with explicit tracks.

=== Style class definition

[source,json]
----
"styles": {
  "display-title": {
    "font":    "serif",
    "size":    "3xl",
    "weight":  "bold",
    "color":   "#1a1a1a",
    "leading": "tight"
  },
  "accent-rule": {
    "color": "#c00",
    "width": "2px"
  },
  "callout": {
    "background": "#fff8e1",
    "radius":     "sm",
    "spacing":    { "inner": "md" }
  }
}
----

Typed properties only — no raw CSS values exposed to the author.
The renderer resolves these to CSS/PDF primitives at export time.

== Persistance

- Content (blocks JSON) → Database
- Export (PDF, HTML)   → Filesystem

== Binding surface

The complete picture

[source,c++]
----
dms_document_save(id?, title, blocks_json, zone_name?)
  → { ok, data: { id, updated_at } }
  Writes to active DB (global or zone). Upsert by id.

dms_document_load(id)
  → { ok, data: { id, title, blocks_json, zone_name, updated_at } }

dms_document_list(zone_name?)
  → { ok, data: Document[] }   -- for a "recent documents" picker

dms_save_pdf(title, zone_name?, export_path?)
  → { ok, data: { path, filename, file_size, exported_at } }
  Follows the 4-step priority. Records to dms_recent_exports.

dms_save_html(title, html_content, zone_name?, export_path?)
  → { ok, data: { path, filename, file_size, exported_at } }

dms_get_recent_exports(limit?)
  → { ok, data: Export[] }    -- ordered by exported_at DESC

dms_open_path(path)           -- open in native viewer (new, Option B)
  → { ok }
----
